## 1. 星海定位

### 1.1 核心定义

星海是鲨之星的**内容广场与集体意识中枢**。它不是一个粉丝论坛，而是一个“有趣内容聚合器”，恰好长在粉丝星球上。

### 1.2 叙事定位

> **“万千星光汇聚，便是我们的海”**

星海是所有鲨鱼的集体意识所汇成的信息海洋。AI先遣队在宇宙各处搜集有趣的信号，鲨鱼们在这里分享、创作、互动，每一次参与都为星球注入💡算力晶体。

### 1.3 体验基调

比刷抖音/小红书更有归属感，但操作门槛不能比它们更高。

核心体验原则：

- **进入零成本**：打开就能刷，不需要关注任何人、不需要设置偏好。AI推荐流天然解决冷启动
- **参与零压力**：点赞最轻，分享链接次之，语音发帖再次之，AI二创最重但也比传统创作轻10倍
- **离开有牵挂**：每次互动都有💡回馈，都在推动星球前进
- **内容永远新鲜**：AI聚合保证即使社区自身UGC产出不足，内容流也不会干涸

### 1.4 阶段演进

| 阶段 | 星海的角色 | 说明 |
|------|-----------|------|
| MVP | 内容广场 + 集体脉搏 | 以AI聚合内容和官方运营为主，UGC为辅 |
| 成长期 | 活跃社区 | UGC占比上升，AI二创成为特色功能 |
| 3D世界开放后 | 入口 + 动态摘要 | 深度互动转移到3D世界，星海变为信息流入口和“宇宙公告板” |

### 2.2 各区域详细说明

**A. 星球脉搏（顶部固定，约15%屏幕）**

一眼扫过的“此刻发生什么”。不是详情页，是氛围感。

展示内容：

- 星球进度百分比 + 今日变化量
- 当前在线/活跃鲨鱼数
- 一句AI动态文案，每隔一段时间自动变化

动态文案示例：

- 日常：“过去1小时有238位鲨鱼发出了星光”
- 赛事期间：“世乒赛决赛中，鲨鱼们的能量贡献暴涨300%”
- 里程碑临近：“距离下一个里程碑只差0.7%，冲！”
- 竞争紧张：“鳗鱼星正在追赶，差距缩小到2.1%”

点击 → 进入星球详情二级页面（完整进度可视化、开发阶段、地标列表、星际排行榜、里程碑时间线）。

**B. 内容标签栏（顶部吸附，横向可滑动）**

| 标签 | 内容来源 | 说明 |
|------|---------|------|
| 推荐 | AI混合推荐 | 默认Tab，所有类型内容智能混排 |
| 热门 | AI爬取 + 站内高互动 | “不局限于偶像”的关键入口，泛娱乐/趣味内容 |
| 星球动态 | 官方运营 | 赛事资讯、里程碑庆祝、剧情更新、活动公告 |
| 创作 | 用户UGC | 原创绘画、文字、视频剪辑、AI二创作品 |
| 赛事 | AI聚合 + 用户 | 赛事期间常驻，平时可隐藏。比赛讨论、实时战报 |

标签可根据运营需要动态增减（如大赛期间临时增加专题标签）。

**C. 内容流（主体，约70%屏幕，无限滚动）**

采用单列卡片流（非双列瀑布），理由：粉丝内容图文混合多，单列阅读体验更好，也更适合评论互动。

**D. 浮动发帖按钮**

右下角浮动“+”按钮。点击后展开发帖选项（详见第4节）。

## 3. 内容体系

### 3.1 内容来源与占比（MVP阶段）

| 来源 | 预估占比 | 说明 |
|------|---------|------|
| AI聚合内容 | 50% | AI先遣队搜集的“宇宙信号”，保证内容密度 |
| 官方运营 | 20% | 赛事、剧情、活动、里程碑 |
| 用户UGC | 20% | 原创帖、分享帖、AI二创 |
| 系统生成 | 10% | 里程碑卡片、成就公告、星际快报 |

### 3.2 AI聚合内容（“AI先遣队信号”）

叙事包装：AI先遣队在宇宙各处截获的信号，带回给鲨鱼们。

**内容处理方式（版权安全）：**

| 方式 | 说明 | 版权风险 |
|------|------|---------|
| 链接卡片 | 只展示标题+缩略图+来源，点击跳转原平台 | 最低 |
| AI摘要 | AI重新组织语言生成简短摘要（<原文10%），附原始链接 | 低 |
| 禁止事项 | 不抓取全文/原图/原视频存储到自己服务器 | — |

MVP阶段统一采用**链接卡片 + AI摘要 + 来源标注**的组合。

AI推荐语示例：

- 🤖 “这场比赛的第四局值得反复看”
- 🤖 “来自地球的趣味信号：今天的互联网被这只猫统治了”
- 🤖 “训练日常曝光，注意看第三张图的细节”

**内容筛选逻辑：**

| 圈层 | 内容类型 | 占比 |
|------|---------|------|
| 核心圈 | 乒乓球/体育相关 | 30% |
| 扩展圈 | 泛娱乐/趣味/热点 | 50% |
| 情感圈 | 粉丝文化/追星相关 | 20% |

### 3.3 卡片类型

**① 图文帖**

- 用户或AI搬运的图文内容
- 封面图 + 文字预览 + 互动栏
- 点击进入详情页展开全文和评论

**② 链接分享卡**

- 从外部平台分享来的内容
- 自动抓取标题 + 封面 + 来源标识
- 用户可附加一句话评论
- 点击跳转原平台

**③ AI二创卡**

- 用户通过AI二创功能生成的作品
- 标注“基于 @XX 的帖子再创”
- 展示原素材 → 二创成果的对比

**④ 里程碑庆祝卡**

- 系统自动生成
- 视觉醒目（特殊边框/动效）
- 全服达成里程碑时插入内容流
- 可点赞和评论，有仪式感

**⑤ 剧情推送卡**

- 新剧情章节解锁时出现
- 点击进入剧情阅读页
- 带有精美封面图和章节预告文字

**⑥ 星际快报卡**

- 竞争事件触发（被超越、反超、差距变化）
- 带有紧迫感的视觉设计
- 号召性文案引导参与

### 3.4 互动体系

每张卡片底部统一互动栏：

| 操作 | 图标 | 说明 |
|------|------|------|
| 点赞 | ❤️ | 最轻量的参与 |
| 评论 | 💬 | 点击进入详情页展开评论区 |
| 分享 | ↗️ | 分享到微信/其他平台 |
| 再创 | ✨ | 图片/视频类内容专属，触发AI二创 |

**评论区设计（初期轻量版）：**

- 平铺式评论，不做树状回复
- 在帖子详情页展开
- 支持文字 + 表情回复
- 后期可升级为树状回复和楼中楼

## 4. 发帖系统

### 4.1 核心原则

**默认状态就够好，用户只需要确认而不需要创作。** 想花心思的人可以编辑，不想的人直接发就行。

### 4.2 发帖入口

点击浮动“+”按钮，展开三个选项：

| 选项 | 图标 | 说明 |
|------|------|------|
| 说点什么 | 🎙️ | 语音/文字发帖 |
| 分享链接 | 🔗 | 粘贴URL快速分享 |
| AI创作 | ✨ | AI辅助生成图文/视频 |

### 4.3 语音发帖流程

```markdown
按住录音 → 松开
  → AI自动转文字 + 润色 + 断句
  → 预览界面：润色后的文字 + 可选配图建议
  → 用户确认（可编辑，也可直接发）
  → 发布 → 信号飞入星海动效 → "+50💡 已到账"
```

全程可在10秒内完成。

### 4.4 链接分享流程

```markdown
方式一：点击"分享链接" → 粘贴URL → 自动解析封面+标题
方式二：从其他App复制链接 → 切回星海 → 自动检测剪贴板 → 弹出"分享到星海？"

→ 可附加一句话（可选，不写也行）
→ 发布 → "+50💡 已到账"
```

两步操作完成。

### 4.5 AI创作辅助

用户给出简单描述或关键词，AI生成图文内容：

- 输入“莎莎世乒赛冠军庆祝” → AI生成星球主题庆祝海报
- 输入“今天心情很好想发个帖” → AI生成配文+配图建议
- 降低创作门槛，让每条帖子视觉质量都不差

### 4.6 发帖收益表

| 行为 | 💡算力晶体奖励 | 说明 |
|------|-------------|------|
| 发布原创帖 | +100💡 | 图文/语音转文字 |
| 分享外部链接 | +50💡 | 带一句话评论额外+20💡 |
| 评论他人帖子 | +10💡 | — |
| 获得他人点赞 | +5💡/个 | 每日上限50💡 |
| 帖子被标记为热门 | +200💡 | 系统自动识别 |
| AI二创作品发布 | +150💡 | 鼓励创作 |

发帖界面轻量展示收益预告，让用户清楚知道“我做这件事有回报”。

## 5. AI二创功能

### 5.1 设计理念

不做独立创作工具，而是**把AI二创嵌入到每一条图片/视频内容上**。将“消费内容”和“创作内容”之间的距离缩到最短。

### 5.2 触发方式

每张图片/视频卡片上有 ✨**“星海再创”** 按钮。点击后进入轻量AI创作界面。

### 5.3 图片二创能力（MVP）

| 功能 | 说明 | 示例 |
|------|------|------|
| 风格转换 | Q版、漫画风、星球主题风、水彩风 | 把比赛照片变成Q版插画 |
| 主题元素叠加 | 添加鲨之星专属元素 | 加鲨鱼、星球背景、粉丝文案框 |
| 文字描述生成 | 用户描述想要的效果，AI执行 | “把背景换成鲨之星的星空” |
| 表情包生成 | 基于图片快速生成表情包 | 截取表情+加文字 |

### 5.4 视频二创能力（后续）

| 功能 | 说明 |
|------|------|
| AI精彩片段提取 | 自动识别高光时刻 |
| 主题BGM + 特效 | 星球主题音乐和视觉特效 |
| 短视频封面生成 | AI生成吸引人的封面图 |

### 5.5 二创社交链

- 二创作品自动关联原内容：“基于 @XX 的帖子再创”
- 原作者收到通知，形成互动
- 在“创作”标签下优先展示二创内容

## 7. 多端策略

### 7.1 产品定位

| 端 | 定位 | 使用场景 |
|----|------|---------|
| App | 完整体验，沉浸留存 | 回家后/休息时间，半小时沉浸 |
| 小程序 | 轻量入口，碎片化+拉新 | 地铁上/碎片5分钟，朋友分享进入 |

### 7.2 用户路径

**小程序引流 → App沉浸**

用户通过小程序接触产品（朋友分享了游戏链接）→ 玩了几天觉得有意思 → 发现发帖、AI二创、3D世界等需要App → 下载App成为深度用户。小程序是漏斗入口，App是留存阵地。

### 7.3 星海功能分端

| 功能 | App | 小程序 |
|------|-----|--------|
| 星球脉搏 | ✅ 完整展示 | ✅ 简化版（进度+在线人数） |
| 内容标签栏 | ✅ 全部标签 | ✅ 仅“推荐”和“热门” |
| 浏览内容流 | ✅ | ✅ |
| 点赞 | ✅ | ✅ |
| 评论 | ✅ | ❌ 显示“在App中参与讨论” |
| 发帖（文字/语音） | ✅ | ❌ 显示“在App中发布” |
| 分享链接 | ✅ | ❌ |
| AI二创 | ✅ | ❌ 显示“在App中解锁再创” |
| 分享到微信 | ✅ | ✅ 强化（小程序天然优势） |
| 星球详情页 | ✅ 完整 | ✅ 简化版 |

### 7.4 小程序引导策略

小程序中星海的关键设计：**看得到但够不到。**

- 能看到精彩的UGC和AI二创作品，但点评论/发帖时提示“在App中解锁”
- 能看到星球详情和排行，但完整的里程碑体验在App中
- 分享到微信的功能被强化，每条内容的分享按钮更醒目
- 分享出去的微信卡片可直接打开小程序浏览，形成拉新闭环

### 7.5 数据打通

**关键原则：两端共享同一个账号和数据体系。**

- 小程序玩游戏获得的⚡、连续天数、宝箱进度，App中无缝衔接
- App中的成就和等级，小程序中也能看到
- 用户感知上是“一个产品的两个入口”而非“两个产品”
