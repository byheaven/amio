# 鲨之星 UI布局设计文档

## 1. 设计总则

### 1.1 设计理念

**“每天打开APP，先看到我们的星球，然后发出今天的光。”**

本产品不是一个游戏工具，是一个粉丝与星球之间的情感连接器。所有UI决策优先服务情感体验，其次才是信息传达。

### 1.2 设计原则

| 原则 | 说明 | 反面案例 |
|------|------|---------|
| 情感优先于数据 | 用视觉讲故事，数字退到需要时才出现 | 首页堆满 ⚡12,450 / 💡6,230 / 💰1280 |
| 一屏一焦点 | 每个页面只做一件核心的事 | 一个页面同时承载游戏、社交、排行、宝箱 |
| 渐进式披露 | 概览在前，详情在深处，想看再看 | 所有数据平铺在表面 |
| 行为最短路径 | 用户每天的核心动作（玩游戏）不超过2次点击 | 游戏入口藏在二级页面 |
| 叙事一致性 | 所有文案、命名、交互都在世界观框架内 | 按钮写“开始游戏”而不是“点亮” |

### 1.3 目标用户画像

以孙颖莎粉丝群体为主：年轻女性为主力，情感驱动型用户，不是硬核游戏玩家。UI语感应温暖、有诗意、低压迫感，避免硬科幻术语。

## 2. 整体架构

### 2.1 底部导航（3个Tab）

| 位置 | 名称 | 图标建议 | 一句话介绍 | 核心职责 |
|------|------|---------|-----------|---------|
| 左 | **星光** | ⭐ 发光的星 | 每一次游戏，都是你发出的一道光 | 游戏、宝箱、每日任务 |
| 中 | **星海** | 🌊 波浪/星群 | 万千星光汇聚，便是我们的海 | 社区动态、星球状态、排行竞争 |
| 右 | **星轨** | 🪐 轨迹环 | 你走过的路，都被星球铭记 | 个人身份、贡献数据、成就记录 |

### 2.2 Tab情感定位

| | 星光 | 星海 | 星轨 |
|--|------|------|------|
| 情感基调 | 安静、专注、使命感 | 热闹、归属、集体感 | 回顾、确认、自豪感 |
| 类比感受 | 独自在窗前点一盏灯 | 走进热闹的广场 | 翻开自己的航行日志 |
| 信息密度 | 极低 | 中等 | 较高 |
| 视觉调性 | 深邃星空，一颗发光的星 | 繁星如海，信息流动 | 光轨线条，个人档案 |

## 3. 星光 Tab — 详细布局

### 3.1 页面定位

用户每日主循环的起点和终点。打开即沉浸，行动路径极短。

### 3.2 用户每日行为流

```markdown
打开APP → 看到星球（情感锚点）
         → 领宝箱（如果有）
         → 点击"点亮"（玩游戏）
         → 完成 → 看到能量飞向星球的动效
         → 可选：挑战Hero模式
         → 离开 / 去星海逛逛
```

### 3.4 各区域详细说明

**A. 顶部状态栏**

| 元素 | 位置 | 说明 |
|------|------|------|
| 🔥连续天数 | 左上 | 如“🔥13”，轻量显示，不加多余文字 |
| ⚙️设置 | 右上 | 进入设置页面 |

不放的内容：货币数量、能源数字、通知图标。保持极简。

**B. 星球氛围区（屏幕上半部 ~45%）**

- 星空是真正的页面背景，从顶部一直延伸，不是一个卡片
- 星球居中偏上，直径约屏幕宽度的40%
- 星球视觉随开发进度变化（详见3.5）
- 星球有呼吸式光晕脉动，给人“活着”的感觉
- 偶尔有小AI机器人从星球表面飘过（3-5秒出现一次，随机方向）
- 星球下方一行文字：“鲨之星 · 已苏醒 67.3%”
- 文字样式：小字号，半透明白色，不抢视觉

**C. 宝箱区（条件显示，~15%）**

三种状态：

| 状态 | 显示内容 | 交互 |
|------|---------|------|
| 宝箱可领取 | 卡片高亮，“✨ 来自星球的回馈”，宝箱图标有光效 | 点击领取，播放开箱动效 |
| 宝箱倒计时 | 小卡片，“物资传送中… 12:34:56” | 不可点击，文字呼吸效果 |
| 无宝箱 | 区域收起，显示一句随机鼓励文案 | 无交互 |

鼓励文案示例：
- “星球记得你的每一份光”
- “AI机器人们正在加班建设中”
- “你的第14天，星球又亮了一点”

**D. 核心行动按钮（视觉焦点）**

| 状态 | 按钮文案 | 视觉 |
|------|---------|------|
| 今日未玩 | **点亮** | 发光状态，呼吸动效，页面视觉焦点 |
| 普通关已完成 | **挑战 Hero** | 颜色变化（如橙→金），暗示更高难度 |
| 今日全部完成 | **明日再来点亮** | 变暗/变柔和，不可点击 |

按钮点击后的过渡动效：点击“点亮” → 一道光从按钮射向星球 → 页面过渡到游戏界面。

**E. 今日预告**

- 按钮下方一行小字
- 展示今日关卡的主题（关联的乒乓球知识点或故事线索）
- 作用：增加每日新鲜感和好奇心
- 示例：“今日主题：那个四块钱的球拍 🏓”

### 3.5 星球视觉演变

星球外观随开发进度自然变化，不需要用户看数字就能感受到“我们在进步”：

| 进度区间 | 星球外观 | 光晕 | 氛围 |
|---------|---------|------|------|
| 0-10% 荒芜期 | 灰暗球体，表面有裂纹 | 微弱、暗红 | 沉寂 |
| 10-25% 萌芽期 | 出现零星绿色光点 | 淡绿色，偶尔闪烁 | 生命初现 |
| 25-50% 建设期 | 绿色区域扩大，出现亮点簇（城市雏形） | 蓝绿色，稳定 | 生机勃勃 |
| 50-75% 繁荣期 | 大面积光带，像地球夜景 | 金蓝色，明亮 | 文明璀璨 |
| 75-99% 启航期 | 星球周围出现环形结构（飞船/空间站） | 强烈白金色 | 激动人心 |
| 100% 登陆期 | 星球全亮，传送门开启 | 彩虹光晕 | 史诗级 |

### 3.6 关键交互动效

| 触发 | 动效 | 时长 | 目的 |
|------|------|------|------|
| 进入星光页 | 星球从远处缓缓靠近 | 0.8s | 仪式感，“回到我的星球” |
| 点击“点亮” | 一道光从按钮射向星球 | 0.5s | 行为→意义的视觉连接 |
| 游戏完成返回 | 能量粒子飞向星球，星球微微变亮 | 1.2s | “我的贡献让星球更亮了” |
| 领取宝箱 | 光从星球落下到宝箱位置 | 0.8s | “来自星球的回馈” |
| 连续天数+1 | 🔥数字跳动，微小庆祝粒子 | 0.5s | 强化连续行为 |

### 4.3 各区域详细说明

**A. 页面标题栏**

| 元素 | 位置 | 说明 |
|------|------|------|
| 🌊 星海 | 左上 | 页面标识 |
| 🔔 通知 | 右上 | 系统通知、回复提醒、星际快报 |

**B. 星球全景卡片（~20%）**

- 星球比星光页更完整，可缓慢自转
- 卡片上浮现三组关键数据：已苏醒百分比、今日在线人数、今日全服进度增量
- 数据以环绕星球的方式排布，不是表格
- 点击卡片 → 进入星球详情页（二级页面）

星球详情页包含（二级页面，不在主布局中）：
- 各开发阶段进度详情
- 已达成里程碑列表
- 已建成地标展示
- 下一里程碑进度和所需能源
- ⚡💡全服能源比例及其对星球的影响

**C. 星际排行横滑卡片（~10%）**

- 横向可滑动的卡片列表
- 每张卡片：星球图标 + 排名 + 名称 + 进度百分比
- 当前星球（鲨之星）卡片高亮，带“我们在这里”标记
- 卡片间距紧凑，一屏可见3-4个
- 竞争状态的情绪暗示：

| 竞争状态 | 视觉暗示 |
|---------|---------|
| 领先 5%+ | 鲨之星卡片金色边框，“遥遥领先” |
| 领先 <5% | 正常高亮 |
| 差距 <2% | 卡片边框变橙色，“紧追不舍” |
| 被超越 | 红色脉动边框，“奋起直追” |

**D. 社区动态流（页面主体，可无限滚动）**

这是星海的核心区域。内容类型混排，按时间线排列：

| 内容类型 | 标识 | 说明 |
|---------|------|------|
| 官方动态 | 📢 | 星球进度通报、活动公告、运营内容 |
| 用户帖子 | 🦈（头像） | 鲨鱼们发的文字/图片内容 |
| 剧情推送 | 📖 | 新章节解锁时出现，点击进入剧情阅读 |
| 星际快报 | 🚨 | 被超越/反超/破纪录等竞争事件 |
| 里程碑庆祝 | 🎉 | 星球达成重大进度时的全服通告 |
| 赛事联动 | 🏆 | 与真实乒乓球赛事相关的特殊内容 |

每条帖子的交互：❤️ 点赞、💬 评论、↗️ 分享

MVP阶段内容策略：
- 社区框架从一开始就搭好
- 初期以官方内容为主（进度通报、剧情推送、运营活动）
- 穿插少量种子用户的UGC内容
- 随用户增长，逐步过渡到UGC为主

**E. 发帖浮动按钮**

- 右下角常驻 ✏️ 按钮
- 点击展开发帖界面
- 发帖后获得 💡算力晶体（叙事化：“向星海投递了一道信号”）

### 4.4 星际快报的特殊呈现

当发生重大竞争事件时，星海页面会有特殊效果：

| 事件 | 入场效果 | 页面变化 |
|------|---------|---------|
| 被其他星球超越 | 顶部下滑红色横幅“🚨 星际警报” | 排行卡片区红色脉动 |
| 反超成功 | 全屏金色粒子飘落 | 星球卡片短暂放大 |
| 打破单日贡献纪录 | 彩色粒子特效 | 动态流置顶庆祝帖 |
| 偶像生日等节日 | 星球周围出现特殊装饰 | 主题色变化 |

### 5.3 各区域详细说明

**A. 身份卡片（~20%）**

- 视觉上像一张“星际身份证”或“登舱凭证”
- 背景可以是半透明的星空渐变卡片
- 核心元素：头像、昵称、当前最高称号、登陆优先级
- 登陆优先级是这张卡片中最醒目的数字，用大字号 + 高亮色
- 点击卡片 → 进入身份详情（编辑头像、查看所有称号、登陆批次预估说明）

**B. 能源双卡片（~15%）**

- 并排两张卡片：⚡动力核心 和 💡算力晶体
- 每张卡片包含：累计总量（大字）、全服排名百分比、本周增量
- 用光谱色带或环形弧线代替粗暴的进度条
- ⚡卡片偏蓝色调，💡卡片偏金色调
- 点击任一卡片 → 进入详细的能源历史（日/周/月的贡献趋势图）

**C. 状态速览（~5%）**

- 一行横排小标签：连续天数、本周任务完成度
- 极简，不展开说明
- 点击“本周任务” → 展开任务列表

**D. 我的星轨 — 时间轴（页面核心，可滚动，~40%）**

这是星轨Tab的灵魂组件。

设计要点：
- 左侧一条竖线，节点用实心圆 ● 表示已达成，空心圆 ○ 表示下一个未达成
- 每个节点标注天数（Day N）和事件描述
- 最新的节点在最上面（倒序），向下滚动看更早的历史
- 最底部是下一个待解锁的里程碑，用虚线和问号暗示“还有更多等你”
- 视觉上带有“光轨”的感觉——已点亮的节点之间有微弱的光线连接

里程碑触发规则：

| 触发类型 | 示例 |
|---------|------|
| 天数类 | Day 1 首次点亮、Day 7 连续一周、Day 30 连续一个月 |
| 成就类 | 首次Hero通关、首次获得钻石宝箱 |
| 贡献量类 | 累计 1,000 / 5,000 / 10,000 / 50,000 能源 |
| 称号获得 | 获得每一个新称号时 |
| 社交类 | 首次发帖、首次被点赞100、邀请首位好友 |

**E. 成就墙（~10%）**

- 横向滚动展示已获得的称号/成就徽章
- 每个徽章是小图标 + 名称
- 末尾显示“+N”暗示还有未解锁的
- 点击 → 进入成就详情页（全部成就列表，已解锁/未解锁状态）

**F. 功能入口（~10%）**

- 底部横排图标按钮：设置、历史贡献明细、帮助/FAQ
- 朴素、功能性，不抢视觉

## 7. 视觉设计方向

### 7.1 色彩体系

| 用途 | 色彩方向 | 说明 |
|------|---------|------|
| 主背景 | 深蓝-深紫渐变 | 星空底色，三个Tab通用 |
| 强调色 | 暖金色 | 按钮、高亮数字、成就 |
| 辅助色1 | 海洋蓝 | ⚡动力核心相关 |
| 辅助色2 | 柔和金 | 💡算力晶体相关 |
| 文字主色 | 白色 / 浅灰 | 在深色背景上保证可读性 |
| 危险/紧迫 | 珊瑚红 | 被超越警报、宝箱即将过期 |

### 7.2 字体与排版

- 标题：圆润的无衬线字体，温暖不冰冷
- 正文：系统默认字体，保证可读性
- 数字：等宽字体，数据对齐整齐
- 整体行距偏大，留白充足，不拥挤

### 7.3 图标风格

- 线性图标为主，线条圆润
- 避免尖锐的科幻感
- 底部导航Tab图标需要带微光效果（选中状态发光）

### 7.4 动效原则

| 原则 | 说明 |
|------|------|
| 有意义 | 每个动效都在讲故事（能量飞向星球 = 你的贡献到达了） |
| 不打断 | 动效不阻碍操作，可以跳过 |
| 轻盈 | 时长控制在0.3s-1.2s之间，不拖沓 |
| 统一方向 | 能量/光 向上飞（从用户到星球），回馈/奖励 向下落（从星球到用户） |
